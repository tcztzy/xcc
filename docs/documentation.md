# Documentation

## MkDocs layout

- The documentation root is `docs/`.
- Site configuration lives in `mkdocs.yml`.
- Navigation is defined in the `nav` section.

## Build commands

- `mkdocs serve` for local preview.
- `mkdocs build` for static site output.

## Writing style

- Use concise, professional English.
- Avoid long prose; prefer checklists and requirements.
- Keep each page focused on a single topic.
