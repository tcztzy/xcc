# Quality

## Linting

- Use `ruff` for linting and formatting.
- All lint checks must pass with zero warnings.

## Typing

- Use `ty` for static type checking.
- Type checking must be clean and strict for all public modules.

## Style discipline

- Keep functions small and composable.
- Avoid implicit globals.
- Prefer explicit data structures and narrow interfaces.
